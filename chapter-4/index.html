Functions & Expressions

Function Declaration:

function printThis() {
console.log('Hello!');
}

Function Expression

const thisWillPrint = () => {
console.log('This will print.');
}

Difference Between Function Declarations and Function Expressions

Function Declarations get hoisted. Function expressions do not.

Concept of Hoisting

With hoisting, the idea is that regardless of where you declare your functions, they get "hoisted" to the top. What this means is you could call your function at the top of your code, but declare your function below it. This is not true of function expressions.

Examples:


printThis();

function printThis() {
    console.log('This will work.');
}


thisWont();

const thisWont = () => {
    console.log('This will not print.');
}

In the example above, the function declaration works because it is hoisted above the fuction call. The function expression does not get hoisted, and thus prints an error to the console. Function expressions tend to be more popular, and the convention is to declare your function expressions at the top of your code.

Default Values

Pass default values by not passing arguments and putting the value as a parameter in the function declaration. Example: 


    const helloThere = (name = "Bob") => {
        console.log(`Hello there ${name}`);
    }
    
    helloThere('Jim');
    helloThere();
    //Console logs "Hello there Jim" the first time. Logs Bob the second time.
